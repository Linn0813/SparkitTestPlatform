# SparkitTestPlatform MVP 范围

## Must（本期交付）

- 用户/项目/成员与 JWT 登录、上下文切换
- 每项目一套可编辑的用例字段模板、缺陷字段模板
- 可编辑缺陷状态；状态变更可触发企业微信机器人 Webhook
- 需求、版本（版本详情关联需求与规划缺陷）
- 功能用例：模块树 + 步骤表格式 CRUD、导入
- 测试计划：关联用例、计划内执行、通过率、相关缺陷
- 缺陷：跟进人、CRUD、关联需求/计划、MinIO 附件、企微通知
- 工作台：版本/缺陷/计划统计图；按角色待办（成员跟进三状态缺陷；测试人员未开始/进行中计划 + 提出人待验收缺陷）
- 项目成员权限：member 可维护需求/版本/模块；用例/计划只读；缺陷可评论与改状态

## Won't（本期不做）

- API/UI/性能测试、JMeter、Kafka
- 多模板/跨项目模板、可配置权限树
- 用例评审、脑图、Excel 导入导出（用例支持 CSV/表格导入，非完整 Excel 套件）
- LDAP/OIDC、通知模板中心
- 缺陷状态流转规则引擎（任意跳转）
- MeterSphere 式可配置工作台卡片、跨项目聚合仪表盘

## 企微

项目设置中配置 Webhook URL；目标状态勾选「通知企微」后，缺陷进入该状态时异步发送 Markdown 消息。
